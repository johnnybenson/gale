250

Built-in Rules

Core Stylelint, SCSS, @stylistic, and order rules. No plugins to install. Same rule names, same config format.

{}

Zero Config Migration

Reads your existing .stylelintrc directly. extends, /* stylelint-disable */, and JSON output all work as-is.

.scss

SCSS & Less

Full preprocessor support. Lint .scss, .less, and .css files with 44 built-in SCSS rules.

--fix

Autofix

Automatically fix problems. Hex shortening, zero units, vendor prefixes, quotes, colon notation, and more.

LSP

Editor Integration

Built-in LSP server for real-time diagnostics. Works with VS Code, Neovim, Helix, Zed, and any LSP client.

||

Parallel & Cached

File-based caching skips unchanged files. Parallel linting via rayon uses all CPU cores.

Get Started

# Install
$ npm install -D @lyricalstring/gale
 
# Lint your project (uses your existing .stylelintrc)
$ npx gale "src/**/*.css"
 
# Autofix problems
$ npx gale --fix "src/**/*.css"

Migrate from Stylelint

One line change. Your config stays exactly the same.

package.json
{
  "scripts": {
−    "lint:css": "stylelint 'src/**/*.css'"
+    "lint:css": "gale 'src/**/*.css'"
  },
  "devDependencies": {
−    "stylelint": "^16.0.0"
+    "@lyricalstring/gale": "^0.1.1"
  }
}
